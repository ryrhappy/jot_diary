# 禁用 Supabase 邮箱验证（开发环境）

如果您在开发环境中不想每次都验证邮箱，可以禁用邮箱验证功能。

## 方法：在 Supabase 后台禁用邮箱验证

### 步骤 1: 进入认证设置

1. 登录 [Supabase Dashboard](https://supabase.com/dashboard)
2. 选择您的项目
3. 点击左侧菜单 **Authentication**（认证）
4. 点击 **Providers**（提供商）标签页

### 步骤 2: 配置邮箱认证

1. 找到 **Email** 提供商
2. 点击 **Email** 卡片进入配置
3. 找到 **"Confirm email"**（确认邮箱）选项
4. **取消勾选** "Confirm email" 选项
5. 点击 **Save**（保存）

### 步骤 3: 验证配置

配置完成后：
- 新用户注册后可以直接登录，无需验证邮箱
- 现有未验证的用户也可以直接登录

## ⚠️ 重要提示

- **开发环境**：建议禁用邮箱验证，方便快速测试
- **生产环境**：强烈建议启用邮箱验证，确保账户安全

## 其他选项

如果您想保留邮箱验证但改善用户体验，可以：

1. **配置邮件模板**：
   - 在 Supabase Dashboard > Authentication > Email Templates
   - 自定义验证邮件的模板和内容

2. **使用开发邮箱服务**：
   - 在开发环境中使用 [Mailtrap](https://mailtrap.io/) 或类似服务
   - 可以查看所有发送的邮件，无需真实邮箱

3. **自动确认邮箱（仅开发环境）**：
   - 在 Supabase SQL Editor 中执行：
   ```sql
   -- 自动确认所有用户的邮箱（仅用于开发环境！）
   UPDATE auth.users 
   SET email_confirmed_at = NOW() 
   WHERE email_confirmed_at IS NULL;
   ```

## 推荐配置

- **本地开发**：禁用邮箱验证
- **测试环境**：可以禁用或使用开发邮箱服务
- **生产环境**：启用邮箱验证 + 自定义邮件模板
