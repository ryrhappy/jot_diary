# 工作过程总结 - Jot Diary PRD 设计

## 系统接收提示词
用户提供了关于 Jot Diary 的初步构思，要求设计一份详尽的 PRD。

## 用户要求
1. 资深产品架构师角色。
2. 围绕极简输入流、存储展示、全文检索、AI 大脑四个核心维度。
3. 文档结构需包含：Vision、Specs、User Flow、Data Schema、AI Tech Path、Non-functional、Constraints。
4. 使用中文输出。

## 修改工作过程
1. **需求分析**：解析用户提出的 Append-Only 模式、STT、RAG、自动化索引等核心理念。
2. **架构设计**：构思流式输入与单一无限长文档的技术实现方式（如虚拟列表）。
3. **编写 PRD**：
   - 定义产品愿景：极简、AI 驱动。
   - 细化功能规格：F1-F11 详细定义。
   - 绘制 Mermaid 流程图。
   - 设计数据库逻辑：包括核心表、实体表及向量存储。
   - 明确 AI 技术路径：Embedding + RAG + NER。
   - 设定设计约束：明确“不做”的功能（无分类、无富文本）。
4. **文档保存**：将 PRD 保存至 `docs/PRD.md`，并根据项目规则生成此总结文档。
