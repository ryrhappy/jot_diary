# 录音界面增加实时计时功能

## 用户需求
用户希望在录音界面能够看到当前的录音时长。

## 修改工作总结
1. **计时逻辑实现**：
   - 在 `InputArea.tsx` 中引入了 `recordingTime` 状态（秒）。
   - 使用 `setInterval` 在录音启动时开始计时，每秒递增。
   - 完善了计时的生命周期管理：在录音结束、取消、确认或发生错误时，确保计时器被正确清除（`clearInterval`）。
2. **UI 界面更新**：
   - 在录音面板的“正在聆听”状态下方增加了时长显示。
   - 使用等宽字体（`font-mono`）展示时间格式 `00:00`，确保计时器跳动时文字位置稳定，不产生抖动。
   - 优化了布局，将时长与状态指示灯对齐，提升视觉层次感。
3. **健壮性优化**：
   - 确保在组件卸载或识别异常中断时，通过 `stopSTT` 统一回收计时器资源，防止内存泄漏。

## 系统接收提示词
“请给录音界面增加录音时长”

## 结论
录音界面现在能够直观地显示用户已录制的时间长度，提升了语音输入的交互反馈质量。
